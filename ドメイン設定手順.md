# kymdesignoffice.com 本番切り替え手順

この手順書は、現在WordPressで表示されている `https://kymdesignoffice.com/` を、新しく作成したGitHub Pagesのホームページに切り替えるためのものです。

専門用語が分からない人でも作業できるように、触る場所と触らない場所を分けて書いています。

---

## まず結論

残っている作業は、主に以下の3つです。

1. ムームードメインの管理画面にログインする
2. DNS設定をGitHub Pages向けに変更する
3. 反映後、GitHub側でHTTPSを有効にする

GitHub側の準備は完了しています。

- GitHub Pagesは公開済み
- `CNAME` は `kymdesignoffice.com` に設定済み
- GitHub PagesのCustom domainも `kymdesignoffice.com` になっています

---

## 今の状態

現在の `kymdesignoffice.com` は、WordPressのサーバーを見に行く設定になっています。

現在の主な設定：

| 項目 | 現在の内容 | 意味 |
|---|---|---|
| ホームページ本体 | `163.44.185.171` | 現在のWordPressサーバー |
| `www`付きURL | `163.44.185.171` | 現在のWordPressサーバー |
| メール | `mx01.lolipop.jp` | メール用の設定 |
| DNS管理場所 | ムームードメイン | 実際に変更する管理画面 |
| ドメイン登録先 | お名前.com | ドメイン契約の登録先 |

今回変更するのは、ホームページの向き先だけです。

メール設定は変更しません。

---

## 作業前に確認すること

作業前に、依頼者へ以下を確認してください。

- 「今のWordPressサイトが本番URLで見られなくなってもよいか」
- 「メールアドレスを使っている場合、メール設定は変更しないことを理解しているか」
- 「ムームードメインのログイン情報があるか」

分からない場合は、ここで止めてください。

---

## 絶対に触らない設定

以下は変更しないでください。

| 種別 | 内容 | 理由 |
|---|---|---|
| MX | `mx01.lolipop.jp` | メールが使えなくなる可能性があります |
| TXT | 既存のTXTレコード | メール認証などに使われる可能性があります |
| SPF / DKIM / DMARC | 既存設定 | メール送信に影響する可能性があります |

「MX」「TXT」「SPF」「DKIM」「DMARC」と書かれているものは、そのまま残してください。

---

## 変更するDNS設定

変更するのは、ホームページ表示に関係するレコードです。

### 1. `kymdesignoffice.com` 本体の設定

現在の `163.44.185.171` を、以下の4つに置き換えます。

| サブドメイン | 種別 | 内容 |
|---|---|---|
| 空欄 または `@` | A | `185.199.108.153` |
| 空欄 または `@` | A | `185.199.109.153` |
| 空欄 または `@` | A | `185.199.110.153` |
| 空欄 または `@` | A | `185.199.111.153` |

### 2. `www.kymdesignoffice.com` の設定

`www` は以下の内容にします。

| サブドメイン | 種別 | 内容 |
|---|---|---|
| `www` | CNAME | `vda8e8ffd7kaezi-lgtm.github.io` |

注意：`www` の向き先には、`/kym-design-office-web/` を付けません。

正しい値：

```txt
vda8e8ffd7kaezi-lgtm.github.io
```

間違い：

```txt
vda8e8ffd7kaezi-lgtm.github.io/kym-design-office-web/
```

## ムームードメインでの作業手順

画面名は変更されることがあります。近い名前のメニューを探してください。

1. ムームードメインにログイン
2. 左メニューまたは管理画面から **ドメイン操作** を開く
3. **ムームーDNS** を開く
4. 対象ドメイン `kymdesignoffice.com` の **変更** または **設定変更** を開く
5. DNSレコードの一覧を確認する
6. `163.44.185.171` が入っているAレコードを探す
7. そのAレコードを、GitHub Pages用の4つのAレコードに置き換える
8. `www` のレコードを、CNAMEで `vda8e8ffd7kaezi-lgtm.github.io` にする
9. MXやTXTは変更せずに保存する

保存前に、下の「保存前チェック」を必ず確認してください。

---

## 保存前チェック

保存前に、以下を確認してください。

- `kymdesignoffice.com` 本体のAレコードが4つある
- 4つのAレコードが以下になっている
  - `185.199.108.153`
  - `185.199.109.153`
  - `185.199.110.153`
  - `185.199.111.153`
- `www` がCNAMEになっている
- `www` の内容が `vda8e8ffd7kaezi-lgtm.github.io` になっている
- MXレコード `mx01.lolipop.jp` を消していない
- TXTレコードを消していない

1つでも不安があれば、保存せずに画面のスクリーンショットを共有してください。

---

## 保存後に起きること

DNSを保存しても、すぐに切り替わらないことがあります。

目安：

- 早い場合：数分
- よくある場合：30分から数時間
- 遅い場合：最大24時間程度

この間、人によって古いWordPressが見えたり、新しいGitHub Pagesが見えたりすることがあります。これはDNS反映待ちなので、しばらく待ってください。

---

## 表示確認

DNS変更後、以下をブラウザで確認します。

```txt
https://kymdesignoffice.com/
https://www.kymdesignoffice.com/
```

期待する状態：

- 新しいホームページが表示される
- `www`付きで開いても、最終的に本体URLへ移動する
- スマホでも表示が崩れていない
- 画像が表示されている
- CONTACTページが開く

古い画面が出る場合は、以下を試してください。

- ブラウザを再読み込みする
- Macの場合：`Command + Shift + R`
- iPhoneの場合：少し時間を置いて再確認する

---

## GitHub側で最後にやること

DNSが反映されたあと、GitHub Pages側でHTTPSを有効にします。

1. GitHubでリポジトリを開く
2. **Settings** を開く
3. 左メニューの **Pages** を開く
4. `Custom domain` が `kymdesignoffice.com` になっていることを確認
5. DNSチェックが成功していることを確認
6. **Enforce HTTPS** にチェックを入れる

`Enforce HTTPS` が押せない場合は、まだDNSまたは証明書の反映待ちです。数時間待ってから再確認してください。

---

## 切り戻し方法

新サイトに問題があり、元のWordPressに戻したい場合は、Aレコードを元の値へ戻します。

戻す値：

```txt
163.44.185.171
```

戻すときも、MXやTXTは触らないでください。

注意：戻してもすぐ全員に反映されるとは限りません。DNSの反映待ちがあります。

## よくある間違い

### `www` の向き先にリポジトリ名を入れてしまう

間違い：

```txt
vda8e8ffd7kaezi-lgtm.github.io/kym-design-office-web/
```

正しい：

```txt
vda8e8ffd7kaezi-lgtm.github.io
```

### MXレコードを消してしまう

メールが使えなくなる可能性があります。MXは触らないでください。

### Aレコードを1つだけ入れる

GitHub Pages用のAレコードは4つ入れてください。

### HTTPSをすぐ有効にしようとする

DNS反映前は失敗することがあります。DNSチェックが成功してから有効にしてください。


この手順書のAレコードとCNAMEの考え方は、GitHub公式ドキュメントに合わせています。
