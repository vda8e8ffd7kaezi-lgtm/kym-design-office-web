# 独自ドメイン本番切り替え手順（kymdesignoffice.com → GitHub Pages）

現在のWordPressサイト（`https://kymdesignoffice.com/`）を停止し、このリポジトリのGitHub Pagesサイトを本番ドメインで表示するための手順です。

このリポジトリ側では、`CNAME` を `kymdesignoffice.com` に設定済みです。DNS管理画面での変更とGitHubへの反映が必要です。

---

## 0. 現状

- 現サイトはWordPressです。
- `kymdesignoffice.com` と `www.kymdesignoffice.com` は現在 `163.44.185.171` を向いています。
- ネームサーバーは `dns01.muumuu-domain.com` / `dns02.muumuu-domain.com` です。
- DNSの実作業はムームードメイン管理画面で行う可能性が高いです。
- ドメイン登録業者はお名前.com（GMO Internet）です。
- メールは `mx01.lolipop.jp` を使っています。MXレコードは触らないでください。

---

## 1. 事前確認

1. GitHubリポジトリがPublicであることを確認
2. GitHub PagesのSourceが `Deploy from a branch` / `master` / `/(root)` であることを確認
3. 現WordPressのバックアップが必要か依頼者に確認
4. `@kymdesignoffice.com` のメールを使っている場合、MXレコードを変更しないことを確認

---

## 2. GitHub側の設定

1. GitHubリポジトリ `kym-design-office-web` を開く
2. **Settings → Pages** を開く
3. **Custom domain** に以下を入力して保存

```txt
kymdesignoffice.com
```

4. DNS反映後、チェックが通ったら **Enforce HTTPS** を有効化

---

## 3. DNSの変更（ムームードメイン）

ムームードメインの管理画面で、`kymdesignoffice.com` のDNSを以下に変更します。

### 変更するレコード

現在の `@` または空欄サブドメインのAレコード `163.44.185.171` は、以下の4つに置き換えます。

| サブドメイン | 種別 | 内容 |
|---|---|---|
| 空欄 または `@` | A | `185.199.108.153` |
| 空欄 または `@` | A | `185.199.109.153` |
| 空欄 または `@` | A | `185.199.110.153` |
| 空欄 または `@` | A | `185.199.111.153` |

`www` はGitHub Pagesへ向けます。

| サブドメイン | 種別 | 内容 |
|---|---|---|
| `www` | CNAME | `vda8e8ffd7kaezi-lgtm.github.io` |

### 触らないレコード

- MXレコード：`mx01.lolipop.jp`
- TXTレコード
- SPF / DKIM / DMARC系のメール認証レコード

---

## 4. 反映確認

DNS反映には数分から最大48時間かかります。

確認コマンド：

```bash
dig kymdesignoffice.com A +short
dig www.kymdesignoffice.com CNAME +short
```

期待値：

```txt
185.199.108.153
185.199.109.153
185.199.110.153
185.199.111.153
```

`www` は以下に向いていればOKです。

```txt
vda8e8ffd7kaezi-lgtm.github.io.
```

---

## 5. 切り戻し方法

問題が起きた場合は、DNSのAレコードを元の値へ戻します。

```txt
163.44.185.171
```

ただし、DNS反映には時間差があります。切り戻し前に、GitHub Pagesの設定画面、DNSレコード、メール影響を確認してください。

---

## 6. 注意

- この切り替え後、既存WordPressサイトは本番ドメインでは表示されなくなります。
- WordPressの契約やデータ自体が消えるわけではありません。
- DNS変更は管理画面ログイン権限が必要です。
- `git push` 後にGitHub Pagesへ反映されます。push前に必ず内容確認してください。
