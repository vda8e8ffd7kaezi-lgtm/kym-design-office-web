# 独自ドメイン設定手順（new.kymdesignoffice.com → GitHub Pages）

既存のWordPressサイト（`https://kymdesignoffice.com/`）はそのまま残し、
この静的サイトを **サブドメイン `new.kymdesignoffice.com`** で**共存**公開するための手順です。
（CNAMEファイルの更新・プッシュなど、コード側の準備は完了済みです）

---

## 0. 前提・確認事項

- **既存WordPressは触りません**：apex `kymdesignoffice.com` のAレコード（`163.44.185.171`）やMXレコードは**そのまま**。WordPressサイトは今まで通り表示されます。
- **追加するのはサブドメインのCNAMEレコード1行だけ**です。
- **ドメイン登録先**：お名前.com（GMO Internet）
- **DNSの管理場所**：ネームサーバーが `dns01/02.muumuu-domain.com` のため、**実際のDNS設定は「ムームードメイン」の管理画面**で行います。
- **メール**：`@kymdesignoffice.com` のメールを使っている場合も、今回はMX・apexを触らないので影響なし。
- **リポジトリ**：GitHub Pages無料利用はリポジトリが **Public** である必要があります。

---

## 1. DNSの変更（ムームードメインの管理画面）

1. ムームードメインにログイン
2. **コントロールパネル → ドメイン操作 → ムームーDNS** を開く
3. 対象ドメイン `kymdesignoffice.com` の「**変更**」をクリック
4. 既存の設定は**削除しない**（apexのA `163.44.185.171` 等はそのまま残す）
5. 「カスタム設定」に**以下の1行だけ追加**：

| サブドメイン | 種別 | 内容 |
|---|---|---|
| `new` | CNAME | `vda8e8ffd7kaezi-lgtm.github.io` |

6. 設定を保存

> 注意：apex（サブドメイン空欄）のレコードは触らないこと。触ると既存WordPressが表示されなくなります。

---

## 2. GitHub側の確認

1. リポジトリ `hp-kym-design-office` → **Settings → Pages**
2. **Source** が `Deploy from a branch` / ブランチ `master` / フォルダ `/(root)` になっているか確認
3. **Custom domain** が `new.kymdesignoffice.com` になっているか確認
   - CNAMEファイルを `new.kymdesignoffice.com` に更新済みなので自動で入る想定。空なら手入力して **Save**
4. DNSが反映されると、Custom domain欄に **緑のチェック（DNS check successful）** が表示される
5. 証明書が自動発行されたら **「Enforce HTTPS」にチェック**を入れる

---

## 3. 反映の確認方法

- DNSの伝播：**数分〜最大48時間**（通常は数十分〜数時間）
- 確認コマンド（ターミナル）：
  ```
  dig new.kymdesignoffice.com +short
  ```
  → `vda8e8ffd7kaezi-lgtm.github.io` と GitHub のIP（`185.199.108.153` など）が返ればOK
- ブラウザで `https://new.kymdesignoffice.com/` を開いて新サイトが表示されるか確認
- 既存の `https://kymdesignoffice.com/`（WordPress）が今まで通り表示されることも確認
- 表示が古い場合はスーパーリロード（Mac: `Cmd + Shift + R`）

---

## 4. うまくいかないときのチェックリスト

- [ ] `new` の CNAME のスペルミス（`vda8e8ffd7kaezi-lgtm.github.io`）はないか
- [ ] apex（空欄）のAレコードを誤って消していないか（消すとWordPressが落ちる）
- [ ] GitHub Pagesの Source が `master /(root)` になっているか
- [ ] GitHub Pages の Custom domain が `new.kymdesignoffice.com` になっているか
- [ ] リポジトリがPublicか
- [ ] HTTPSのチェックは、緑チェックが出て証明書発行が終わってから（早すぎると失敗する。最大24時間待つ）

---

## 5. 将来このサイトをメイン（apex）に切り替える場合

WordPressをやめてこの静的サイトを `kymdesignoffice.com` 本体にする場合は、
apexのAレコードをGitHubの4つ（`185.199.108〜111.153`）に変更し、CNAMEファイルを `kymdesignoffice.com` に戻します。
※その時点でWordPressは表示されなくなるため、事前にバックアップを取ること。

---

## メモ

- GitHub PagesのIPアドレスは GitHub公式の値です。公式ドキュメント「Managing a custom domain for your GitHub Pages site」で最新確認可。
- 設定変更後に詰まったら「`dig new.kymdesignoffice.com +short` の結果」「GitHub Pages設定画面のスクショ」を共有してもらえれば一緒に確認できます。
